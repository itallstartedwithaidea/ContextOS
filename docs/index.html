BUILT ON

Six giants.
Unified.

ContextOS was built with respect for every project that came before it. Here is what each one contributed — and what was missing.

modelcontextprotocol/servers
★ 80.5k
The MCP protocol standard. ContextOS is 100% compatible and speaks MCP natively.
Missing: Memory, retrieval, planning, orchestration — everything above the protocol layer.
infiniflow/ragflow
★ 74.4k
Production-grade RAG engine with deep document understanding and agent capabilities.
Missing: Cross-session state, staleness detection, multi-corpus routing, feedback loop.
dair-ai/Prompt-Engineering-Guide
★ 71.3k
The definitive prompt engineering corpus. Powers ContextOS's planning layer spec templates.
Missing: Runtime integration, prompt versioning, outcome measurement.
upstash/context7
★ 48.2k
Live, version-specific library documentation injection for LLMs. Elegant solution to the stale-docs problem.
Missing: Memory layer, caching, session continuity — purely stateless.
thedotmack/claude-mem
★ 33.5k
Captures and compresses Claude coding sessions. 33k stars = real developer pain around agent memory.
Missing: Memory dies with the session. No persistence, tiering, entity graph, or conflict resolution.
ComposioHQ/composio
★ 27.3k
1000+ tool integrations with managed auth. The best solution for agent-to-API connectivity.
Missing: Tool chaining, output caching, retry policies, version pinning, cost tracking.
gsd-build/get-shit-done
★ 26.5k
Spec-driven development system for Claude Code. Forces structure before execution.
Missing: Dynamic plan revision, constraint propagation, spec versioning, outcome evaluation, Pre-Response Sparring Hook.
UNIQUE TO CONTEXTOS

The Pre-Response
Sparring Hook.

Before any agent output fires, ContextOS asks: do I have enough information to act on this correctly, or am I just pattern-matching? No other repo in this space has this.

plan_spar tool · pre-response evaluation
REQUEST
"Delete all campaigns with ROAS below 2.0"
CONTEXT
Q4 budget planning session
VERDICT
HOLD
REASON
Operation is irreversible. 23 campaigns would be deleted. No confirmation that brand campaigns are excluded from the ROAS threshold.
WARNING
Pattern match: "low-ROAS cleanup". Q4 pre-holiday context may make these campaigns strategic, not inefficient.
ACTION
Request confirmation with campaign list preview before executing.
